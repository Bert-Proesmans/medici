The state machine.

The developer is encouraged to alter this contents of this structure to fit his use case. Each state machine MUST have a state, history and transaction field AT MINIMUM.

Safety

Fields state and history are not accessible because altering them would cause the state transition and -history system to fall apart. Any code that has valid reason to alter these fields should be defined within this crate.

Fields

transaction: X::Transaction

Field to store the provided Transaction object as rquired by the current state.

triggers: TriggerService<TimingItem, TriggerItem>

Object for manipulating Triggers.

entities: EntityService<Entity>

Object for manipulating Entitys.

transactions: TransactionStorage<TransactionItem>

Storage object allowing [PushdownInto] and [PullupInto] to store the [Transaction] objects for each state to be re-used.
